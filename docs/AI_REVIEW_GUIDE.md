# BobFull AI PR 1차 리뷰 가이드

## 1. 역할

PR 1차 리뷰는 PR 작성자가 아닌 다른 팀원의 AI 에이전트가 수행한다.

사용 명령:

```text
PR #번호 1차 리뷰하라
```

구현 AI의 자체 Diff 검토는 1차 리뷰로 계산하지 않는다. AI 리뷰는 Human Merge 판단을 대체하지 않는다.

## 2. 리뷰 시작 조건

### 2.1 최소 시작 조건

다음 세 가지를 실제 GitHub 기준으로 확인할 수 있으면 1차 AI 리뷰를 수행한다.

1. 최신 Head Commit SHA
2. 최신 Head의 실제 Diff
3. 해당 PR이 구현하려는 Issue 또는 작업 범위

작업 범위는 최종 Issue 계약을 우선한다. Issue가 미완료이거나 최종화되지 않았으면 PR 본문과 확인 가능한 문서를 기준으로 리뷰하고, 계약 미확정을 발견 사항으로 기록한다.

### 2.2 함께 확인할 자료

가능한 범위에서 다음을 함께 확인한다.

- 최종 Issue 계약
- 구현 AI가 Issue에 남긴 `HUMAN_APPROVED` 승인 기록
- PR 본문
- 관련 코드와 테스트
- 테스트·build·직접 검증·CI 증거
- 미검증 범위와 남은 위험
- 담당자의 PR Human 이해도 답변
- Human 리뷰어가 작성한 `Human 리뷰`

다음 상태는 1차 리뷰 중단 사유가 아니다.

- PR이 Draft 또는 구현 중 상태임
- Issue가 최종화되지 않았거나 상태 기록이 불완전함
- 구현 승인 기록이 없거나 확인되지 않음
- 테스트·build가 실패함
- 테스트·직접 검증·CI가 실행되지 않음
- 결과에 `FAIL`, `HOLD`, `NOT_RUN`이 있음
- Human 이해도 답변이 없거나 `모름`, 불완전한 답변이 있음
- Human 리뷰가 아직 작성되지 않음
- PR 설명과 실제 Diff가 일치하지 않음

이러한 항목은 리뷰 불가 사유가 아니라 `BLOCKER`, `MAJOR`, `MINOR`, `QUESTION` 또는 미검증 범위로 기록할 대상이다.

다음 경우에만 `1차 리뷰 준비 미완료`로 보고한다.

- 최신 Head SHA를 확인할 수 없음
- 최신 Head의 실제 Diff에 접근할 수 없음
- Issue와 PR 본문 모두에서 작업 범위를 확인할 수 없어 판단 기준을 세울 수 없음

일부 자료가 없으면 확인 가능한 범위까지 리뷰하고 확인하지 못한 내용을 명시한다. 1차 AI 리뷰 완료는 PR 구현 완료, 테스트 통과 또는 Merge 가능을 의미하지 않는다.

## 3. 검토 항목

기본 검토:

- Issue 범위·제외 범위·완료 조건
- Issue 최종 계약과 구현 승인 기록의 연결
- 승인 이후 계약 변경 여부
- 요청부터 응답·저장까지 실제 코드 흐름
- 입력 검증과 예외 처리
- 코드와 PR 설명의 일치
- 테스트와 완료 조건의 연결
- 실제 실행 결과와 미검증 범위
- 범위 밖 변경과 불필요한 복잡성
- 비밀정보·개인정보 포함 여부

강화 검토:

- 인증·인가·소유권
- 트랜잭션·Rollback·부분 성공
- 상태·금액·수량 정합성
- 중복 요청·멱등성
- 동시성·락·재시도
- 외부 연동 실패와 보상

구현 승인 기록은 Human이 최종 계약을 확인한 뒤 같은 구현 명령을 입력했고, 구현 AI가 이를 GitHub Issue에 기록했다는 절차 증거다. 기록이 없거나 승인 이후 계약이 변경됐더라도 1차 리뷰는 계속하고 절차 위험으로 기록한다.

## 4. Human 입력 검토

### 4.1 담당자 Human 이해도 답변

리뷰 AI는 담당자 답변이 있으면 최신 코드와 비교한다.

- 정확하면 `일치`로 기록한다.
- 설명이 부족하면 답변을 덮어쓰지 않고 `AI 보완 설명`을 작성한다.
- 잘못된 내용은 `보완 필요`로 표시하고 실제 코드 근거를 제시한다.
- 답변이 없으면 `미작성`으로 기록하되 코드·테스트 리뷰는 계속한다.
- Human 답변과 AI 보완 설명을 반드시 구분한다.
- 점수·합격·불합격·모범 답안 형식은 사용하지 않는다.

Human 답변이 리뷰 이후 작성되면 최신 Head와 답변을 함께 다시 확인한다. 답변 보완 확인은 Merge 전 재검토 항목이며 1차 코드 리뷰의 시작 조건이 아니다.

### 4.2 Human 리뷰어 리뷰

Human 리뷰어가 PR 템플릿의 `Human 리뷰`를 작성했으면 실제 Diff와 다음 내용을 대조한다.

- `내가 이해한 구현 흐름`이 실제 요청·처리·저장·응답 흐름과 일치하는가
- `이해되지 않거나 추가 설명이 필요한 부분`을 코드 근거로 설명할 수 있는가
- `문제로 보이거나 다시 확인할 부분`이 실제 결함·테스트 공백·계약 충돌인가

예를 들어 로그인 PR이라면 Controller 진입, Service의 회원·비밀번호·활성 상태 검증, 토큰 발급 흐름이 실제 코드와 일치하는지 확인한다. 회원 없음과 비밀번호 불일치 응답의 동일 여부, 발급 토큰으로 보호 API에 접근하는 테스트 존재 여부처럼 리뷰어가 제기한 확인 사항도 실제 예외 매핑과 테스트 코드에서 검증한다.

Human 리뷰의 원문은 덮어쓰지 않는다. 단순 이해 질문은 설명 대상으로 정리하고, 실제 결함 근거가 있으면 발견 사항에 연결한다. 같은 내용이 AI 발견 사항에 이미 있으면 중복 지적하지 않고 연결 관계만 기록한다.

Human 리뷰가 아직 없으면 `미작성`으로 기록할 수 있으나 1차 AI 리뷰는 계속한다. AI 리뷰가 Human 리뷰를 대신하지 않는다.

Human 리뷰에 리뷰어·기준 Head SHA·재확인 상태를 추가하는 방안과 PR 본문 대신 GitHub Review·댓글에 작성하는 방안은 별도 운영 결정 사항이다. 현재 양식에는 적용하지 않는다.

## 5. 발견 사항 기준

실제 코드 근거가 있을 때만 발견 사항을 남긴다.

- **BLOCKER**: 보안·데이터·정합성·핵심 계약 훼손 위험
- **MAJOR**: 요구사항 실패·높은 확률의 런타임 오류·필수 실패 처리 누락
- **MINOR**: 제한된 유지보수성·명확성·테스트 공백
- **QUESTION**: 결함으로 단정할 수 없고 Human 결정이 필요한 사항

근거 없는 추측, 개인 취향, Issue 밖 미래 기능, 무관한 리팩터링은 지적하지 않는다.

발견 사항 형식:

```markdown
### [MAJOR] 구체적인 제목

- 위치:
- 확인된 사실:
- 실패 시나리오:
- 영향:
- 최소 수정 방향:
- 확인할 검증:
```

## 6. 리뷰 결과 작성

리뷰 AI는 PR 댓글 또는 리뷰에 다음을 남긴다.

```markdown
## AI 1차 리뷰 요약

- 기준 Head:
- 결론: 수정 필요 | 확인 필요 | 특이사항 없음
- 확인한 코드 흐름:
- 확인한 테스트·검증:
- 확인하지 못한 범위:

## Issue 계약·승인 기록

- 최종 Issue 계약 상태:
- 구현 승인 기록 상태: 확인됨 | 미확인 | 없음
- 승인 이후 계약 변경 여부:

## Human 답변 검토

### 질문 1
- Human 답변 상태: 일치 | 보완 필요 | 미작성
- AI 보완 설명:

## Human 리뷰 확인

- Human 리뷰 상태: 작성됨 | 미작성
- 이해한 흐름과 실제 코드 일치 여부:
- 추가 설명이 필요한 부분:
- 발견 사항과 연결되는 문제:

## 발견 사항

### [심각도] 제목
...

## 구현 AI 수정 경계

- 자동 수정 가능한 항목:
- Human 결정이 필요한 항목:
```

리뷰 AI는 PR 양식의 체크리스트나 승인 상태를 Merge 조건처럼 사용하지 않는다. PR 담당자는 이해도 답변을 작성하고, Human 리뷰어는 `Human 리뷰`를 작성한다. 둘 중 하나가 미작성이어도 코드 리뷰는 진행하며 상태와 위험을 기록한다.

## 7. 리뷰 단계 구분

- **1차 AI 리뷰**: 현재 Head의 결함·계약 충돌·누락·미검증 위험을 찾는다.
- **수정 후 재리뷰**: 이전 지적과 Human 리뷰 문제의 해결 여부, 새 결함을 최신 Head에서 확인한다.
- **Merge 전 최종 재검토**: 테스트·검증·Human 답변·Human 리뷰·CI·승인 조건과 남은 위험을 확인한다.

1차 리뷰에서 `수정 필요` 또는 `확인 필요`가 나와도 리뷰는 정상적으로 완료된 것이다. 다만 해당 PR은 Merge 준비가 완료된 것이 아니다.

## 8. 리뷰 경계

리뷰 AI는 다음을 수행하지 않는다.

- 코드·문서 수정
- Commit·Push
- PR 본문 임의 수정
- Human 답변 또는 Human 리뷰 대리 작성
- Approve
- Merge

정책·API·DB·상태·권한·트랜잭션 재결정이 필요한 사항은 QUESTION으로 남긴다.
