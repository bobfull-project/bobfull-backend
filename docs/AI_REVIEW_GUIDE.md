# BobFull 담당자 AI PR 검토·리뷰 반영 가이드

## 1. 역할

이 문서는 PR 담당자의 AI가 자신의 연결 PR을 검토하고, 담당자 Human 답변을 보완하며, PR에 등록된 리뷰·댓글의 반영 여부를 판단할 때 사용하는 기준이다.

별도의 다른 팀원 AI 리뷰는 BobFull 워크플로우의 공식 단계나 필수 조건으로 관리하지 않는다. PR에 리뷰·댓글이 등록되면 작성 주체와 관계없이 참고 입력으로 읽는다.

담당자 AI의 검토는 다음을 수행한다.

- 최신 Head의 실제 Diff 자체 검토
- Issue 계약·승인 기록·PR 설명 정합성 확인
- 테스트·build·직접 검증 증거 확인
- 담당자 Human 이해도 답변 검증과 `AI 보완 설명` 작성
- Human 리뷰 확인
- 등록된 리뷰·댓글의 근거 확인과 범위 안 수정

담당자 AI는 Approve와 Merge를 수행하지 않으며 Human 코드 리뷰를 대체하지 않는다.

## 2. 실행과 시작 조건

연결된 PR이 존재하는 Issue에 다음 명령을 사용한다.

```text
Issue #번호 구현하라
```

담당자 AI는 Issue와 PR 상태를 읽고 PR 검토·리뷰 반영 단계부터 재개한다.

### 최소 검토 조건

다음 세 가지를 확인할 수 있으면 현재 상태와 관계없이 검토한다.

1. 최신 Head Commit SHA
2. 최신 Head의 실제 Diff
3. 해당 PR의 Issue 또는 작업 범위

다음 상태는 검토 중단 사유가 아니다.

- PR이 Draft 또는 구현 중임
- Issue 계약·승인 기록이 미완료임
- 테스트·build가 실패함
- 테스트·직접 검증·CI가 미실행임
- `FAIL`, `HOLD`, `NOT_RUN`이 존재함
- 담당자 Human 답변이 미작성·불완전·`모름`임
- Human 리뷰가 미작성 또는 이전 Head 기준임
- 다른 리뷰·댓글이 없음
- PR 설명과 실제 Diff가 불일치함

이러한 항목은 현재 결함·확인 필요·미검증 위험으로 기록한다.

최신 Head SHA, 실제 Diff 또는 작업 범위를 확인할 수 없을 때만 검토 준비 미완료로 보고한다.

## 3. 검토 입력

가능한 범위에서 다음을 확인한다.

- 최종 Issue 계약
- 담당자 AI가 Issue에 남긴 `HUMAN_APPROVED` 기록
- 승인 이후 계약 변경 여부
- PR 본문
- 최신 실제 Diff
- 관련 코드와 테스트
- 테스트·build·직접 검증·CI 증거
- 미검증 범위와 남은 위험
- 담당자 Human 이해도 답변
- Human 리뷰
- PR의 Review·Conversation 댓글·inline comment

리뷰·댓글의 작성자가 Human인지 AI인지에 따라 우선순위를 자동으로 정하지 않는다. 실제 코드·계약·테스트 근거로 판단한다.

## 4. 코드와 계약 검토

### 기본 검토

- Issue 범위·제외 범위·완료 조건
- Issue 최종 계약과 구현 승인 기록의 연결
- 승인 이후 계약 변경 여부
- 요청부터 응답·저장까지 실제 코드 흐름
- 입력 검증과 예외 처리
- 코드와 PR 설명의 일치
- 테스트와 완료 조건의 연결
- 실제 실행 결과와 미검증 범위
- 범위 밖 변경과 불필요한 복잡성
- 비밀정보·개인정보 포함 여부

### 강화 검토

인증·결제·예약·좌석·상태 변경·정산·외부 연동에는 다음을 추가한다.

- 인증·인가·소유권
- 트랜잭션·Rollback·부분 성공
- 상태·금액·수량 정합성
- 중복 요청·멱등성
- 동시성·락·재시도
- 외부 연동 실패와 보상

## 5. 담당자 Human 이해도 검토

담당자 Human은 PR 본문의 `담당자 답변`을 직접 작성한다.

담당자 AI는 각 답변을 최신 코드와 비교하고 다음만 작성한다.

```markdown
**AI 답변 검토:** `일치 | 보완 필요 | 미작성`

**AI 보완 설명:**
```

판단 기준:

- 정확함: `일치`
- 핵심 흐름·예외·테스트 설명이 부족함: `보완 필요`와 `AI 보완 설명`
- 실제 코드와 다름: `보완 필요`와 코드 근거
- 답변 없음: `미작성`

담당자 AI는 Human 답변 원문을 수정하거나 대신 작성하지 않는다. 점수·합격·불합격·모범 답안 형식은 사용하지 않는다.

다른 리뷰어 또는 다른 AI는 이 Human 이해도 답변을 보완하는 공식 역할을 갖지 않는다. 해당 보완은 담당자 AI의 책임이다.

## 6. Human 리뷰 확인

Human 리뷰어는 PR 본문에 다음 메타데이터와 리뷰 내용을 작성한다.

- 리뷰어
- 기준 Head SHA
- 리뷰 시각
- 수정 후 재확인: `필요 | 완료 | 해당 없음`
- 내가 이해한 구현 흐름
- 이해되지 않거나 추가 설명이 필요한 부분
- 문제로 보이거나 다시 확인할 부분

담당자 AI는 다음을 확인한다.

- 기준 Head SHA가 현재 Head와 같은가
- 다르면 수정 후 재확인이 `필요`로 표시됐는가
- 이해한 흐름이 실제 코드 흐름과 일치하는가
- 추가 설명 요청을 코드 근거로 설명할 수 있는가
- 문제 제기가 실제 결함·테스트 공백·계약 충돌인가

담당자 AI는 Human 리뷰 원문을 덮어쓰지 않는다. 새 Commit으로 Head가 바뀌면 이전 리뷰가 최신 코드 검토를 증명한다고 간주하지 않는다.

## 7. PR 리뷰·댓글 판단

PR에 등록된 모든 리뷰·댓글은 선택적 참고 입력이다. 특정 리뷰가 있어야 검토나 수정 단계가 시작되는 것은 아니다.

각 의견은 다음으로 분류한다.

### 명확한 결함

조건:

- 실제 코드·Issue 계약·테스트 근거가 있음
- Issue 범위 안에서 수정 가능함
- 정책 재결정이 필요하지 않음

처리:

- 수정
- 영향받는 테스트·build·직접 검증 재실행
- PR 기록 갱신

### 설명 또는 확인 요청

조건:

- 결함으로 단정되지 않음
- 코드 흐름이나 테스트 범위 설명이 필요함

처리:

- 확인 결과와 코드 근거 기록
- 임의 코드 수정하지 않음

### Human 결정 필요

조건:

- 정책·API·DB·상태·권한·트랜잭션 재결정
- 다른 담당자 계약 변경
- 범위 확장 또는 새로운 기술 도입

처리:

- 수정하지 않음
- Human 결정 요청

### 미반영

조건:

- 근거를 확인할 수 없음
- Issue 범위 밖임
- 이미 해결됐거나 현재 Head에 적용되지 않음

처리:

- 반영하지 않은 이유 기록

## 8. 발견 사항 기준

실제 근거가 있을 때만 발견 사항으로 기록한다.

- **BLOCKER**: 보안·데이터·정합성·핵심 계약 훼손 위험
- **MAJOR**: 요구사항 실패·높은 확률의 런타임 오류·필수 실패 처리 누락
- **MINOR**: 제한된 유지보수성·명확성·테스트 공백
- **QUESTION**: 결함으로 단정할 수 없고 Human 결정이 필요한 사항

근거 없는 추측, 개인 취향, Issue 밖 미래 기능과 무관한 리팩터링은 지적하지 않는다.

## 9. 담당자 AI 검토 결과 기록

PR 본문의 `담당자 AI 검토·수정 기록`에 다음을 작성한다.

```markdown
## 담당자 AI 검토·수정 기록

- 담당자 AI 검토 기준 Head:
- 담당자 AI 검토 결과: `수정 필요 | 확인 필요 | 특이사항 없음`
- 확인한 PR 리뷰·댓글:
- 반영한 항목:
- 반영하지 않은 항목과 이유:
- Human 결정이 필요한 항목:
- 재실행 테스트·build 결과:
- 최신 Head SHA:
- 남은 미검증 위험:
```

검토 결과가 `특이사항 없음`이어도 확인 범위와 미검증 위험을 기록한다. 이 결과는 독립 승인이나 Merge 가능 판정이 아니다.

## 10. 최신 Head 재검토

수정 Commit이 올라오면 이전 결과를 그대로 재사용하지 않는다.

- 현재 Head SHA와 수정 Diff 확인
- 기존 지적 해결 여부 확인
- 영향받은 테스트·build·직접 검증·CI 확인
- 새 결함과 PR 설명 불일치 확인
- 담당자 Human 답변과 AI 보완 설명 재대조
- Human 리뷰 기준 Head와 재확인 상태 확인
- 남은 PR 리뷰·댓글 확인

기준 Head가 달라진 Human 리뷰는 재확인 전까지 최신 코드 리뷰 완료로 계산하지 않는다.

## 11. Merge 전 경계

다음이 충족돼야 담당자 AI가 `FINAL_HUMAN_REVIEW`로 보고할 수 있다.

- 담당자 Human 답변과 AI 검토 완료
- Human 리뷰의 최신 Head 재확인 상태 확인
- 필수 테스트·build·직접 검증 결과 확인
- 최신 Head 기준 담당자 AI 검토 완료
- 해결되지 않은 BLOCKER 없음
- 남은 Human 결정 필요 사항 명시

GitHub Rules의 Approve 조건과 Merge 결정은 Human이 담당한다.

## 12. 금지 사항

담당자 AI는 다음을 수행하지 않는다.

- Human 답변 또는 Human 리뷰 대리 작성
- 근거 없는 리뷰 의견 자동 반영
- 정책·API·DB·권한·트랜잭션 임의 결정
- Approve
- Merge
