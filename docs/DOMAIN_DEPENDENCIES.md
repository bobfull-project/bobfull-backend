# BobFull 도메인 의존성과 변경 영향

## 1. 목적

이 문서는 정책이나 상태 전이를 새로 정의하지 않는다.

예약·좌석·결제·취소·환불·방문·노쇼·채팅·지급 예정 예약금이 서로 어떻게 연결되는지 확인하고, 한 영역의 변경이 다른 담당 영역과 문서·테스트에 누락되지 않도록 사용하는 변경 영향 기준이다.

정책이 이 문서와 충돌하면 최신 프로젝트 확정 컨텍스트와 팀 결정이 우선한다. 확정되지 않은 정책은 이 문서에서 임의로 결정하지 않는다.

전체 사용자 흐름은 [`bobfull_full_flowchart_mermaid.md`](./bobfull_full_flowchart_mermaid.md)를 확인한다.

## 2. 도메인 연결 요약

```text
회원·인증
→ 식당·사장님 권한
→ 합석 테이블·예약 회차
→ 예약 생성·추가 참여
→ 개인별 예약금 결제
→ 참여 인원·예약 상태 반영
→ 취소·환불
→ 사장님의 참여자별 방문·노쇼 처리
→ 지급 예정 예약금 조회

예약 참여 관계
→ 예약 참여자 전용 채팅 권한
```

| 도메인 | 선행 입력 | 주요 결과 | 직접 영향을 받는 도메인 | 핵심 담당 |
|---|---|---|---|---|
| 회원·인증 | 회원 정보, 권한 | 인증 사용자와 역할 | 식당, 예약, 채팅, 관리자 | 정용태 |
| 식당·사장님 | 인증된 사장님 | 본인 식당과 소유권 | 테이블·회차, 노쇼, 지급 예정금 | 정용태 |
| 테이블·예약 회차 | 식당, 테이블 정원, 날짜·시간 | 예약 가능한 테이블·시간 | 예약, 좌석 재고, 검색 | 김홍기 |
| 예약·참여자 | 회차, 사용자, 결제 성공 | 예약·참여자·참여 인원·상태 | 좌석, 취소, 채팅, 노쇼, 지급 예정금 | 배지현 |
| 좌석·동시성 | 테이블 정원, 유효 참여 인원 | 추가 참여 가능 여부와 정원 보호 | 예약, 결제 실패 보상 | 배지현 |
| 결제 | 사용자, 예약 또는 회차 | 개인별 `PAID/FAILED/REFUNDED` | 예약 등록, 취소, 환불, 지급 예정금 | 김현승 |
| 취소·환불 | 예약·참여자 상태, 마감 시각, 귀책 | 참여자·예약·결제 상태 변경 | 좌석, 회차 복구, 채팅, 지급 예정금 | 정용태·김현승·배지현 |
| 방문·노쇼 | 사장님 소유권, `RESERVED` 참여자 | `VISITED/NO_SHOW`, 예약 종료 판단 | 지급 예정금, 노쇼율, 관리자 조회 | 정용태·김현승 |
| 채팅 | 인증 사용자와 유효 예약 참여 관계 | 예약별 메시지 접근 권한 | 예약 취소·종료 | 김현승·배지현 |
| 지급 예정 예약금 | 결제·환불·취소·방문·노쇼 결과 | 사장님 조회용 예상 금액 | 사장님 권한, 관리자 조회 | 김현승 |

담당자는 단독 소유권을 뜻하지 않는다. 여러 도메인이 연결되는 Issue는 관련 담당자가 계약과 실패 결과를 함께 확인한다.

## 3. 핵심 공동 작업 경계

### 예약 생성

```text
테이블·회차 유효성 확인
→ 모집 마감 확인
→ 개인 예약금 결제
→ 결제 성공
→ 예약 생성
→ 최초 참여자 등록
→ 참여 인원 1명
→ RECRUITING
```

필수 공동 검토:

- 김홍기: 테이블·회차 유효성
- 김현승: 결제 성공·실패 결과
- 배지현: 예약·참여자·인원 반영과 중복 생성 방지

### 추가 참여

```text
추가 참여 가능 여부 확인
→ 개인 예약금 결제
→ 결제 성공
→ 참여자 등록
→ 참여 인원 증가
→ 예약 상태 재계산
```

필수 공동 검토:

- 배지현: 정원·동시성·상태 재계산
- 김현승: 결제와 실패 보상
- 김홍기: 조회 조건과 회차 정보

### 취소·환불

```text
취소 가능 조건과 귀책 판단
→ 참여자 상태 변경
→ 환불 또는 미환불
→ 유효 참여 인원 재계산
→ 예약 상태 재계산
→ 필요 시 테이블·회차 복구
→ 채팅 권한과 지급 예정금 반영
```

필수 공동 검토:

- 정용태: 취소 정책과 예약 완료 처리
- 김현승: 환불·미환불·지급 예정금
- 배지현: 참여 인원·예약 상태·좌석 반영
- 김홍기: 테이블·회차 복구

### 방문·노쇼와 예약 종료

```text
사장님 소유권 확인
→ 예약 참여자 목록 조회
→ RESERVED 참여자를 VISITED 또는 NO_SHOW로 변경
→ 모든 참여자 처리 여부 확인
→ 예약 CLOSED 전환
→ 지급 예정 예약금과 노쇼율 반영
```

방문 코드와 사용자 직접 체크인은 사용하지 않는다.

## 4. 정책 변경 영향표

| 변경 정책 | 필수 영향 도메인 | 함께 확인할 항목 |
|---|---|---|
| 테이블 정원 `2·4·8` 변경 | 테이블, 예약, 좌석, 검색 | 정원 검증, 남은 참여 인원 계산, 동시성 테스트, 조회 응답 |
| 최소 성사 인원 `2명` 변경 | 예약, 결제, 취소, 환불, 모집 마감 | `RECRUITING/CONFIRMED` 기준, 모집 실패, 환불 대상 |
| 모집 마감 `시작 2시간 전` 변경 | 예약, 결제, 취소, 환불, 채팅, 알림 | 생성·참여 차단, 재모집, 취소 환불 기준, 채팅 쓰기 권한 |
| 결제와 참여 등록 순서 변경 | 결제, 예약, 좌석 | 결제 성공 후 저장 실패, 중복 요청, 보상 처리, 트랜잭션 경계 |
| 예약 상태 기준 변경 | 예약, 취소, 채팅, 노쇼, 지급 예정금 | 상태 전이, 접근 권한, 종료 조건, 테스트 |
| 참여자 상태 기준 변경 | 예약, 취소, 노쇼, 채팅, 지급 예정금 | 유효 참여 인원, 접근 권한, 노쇼율, 금액 계산 |
| 최초 예약자 취소 정책 변경 | 취소, 예약, 환불, 좌석, 회차, 채팅 | 전체 취소 여부, 나머지 참여자 환불, 회차 복구 |
| 환불 기준 변경 | 결제, 취소, 지급 예정금, 관리자 | 환불 상태, 귀책, 미환불 금액 포함 여부 |
| 노쇼 처리 주체·시점 변경 | 인증, 사장님, 예약, 참여자, 지급 예정금 | 소유권, 허용 상태, 종료 조건, 감사 이력 |
| 채팅 접근 조건 변경 | 인증, 예약, 참여자, 취소 | 입장·조회·전송 권한, 취소·종료 후 권한 |
| 지급 예정금 계산 변경 | 결제, 환불, 취소, 노쇼, 사장님 | 포함·제외 항목, 조회 시점, 테스트 데이터 |

## 5. 문서와 구현 변경 체크리스트

정책·API·DB·상태를 변경하는 Issue는 필요한 항목만 확인한다.

- [ ] 최신 프로젝트 확정 컨텍스트와 충돌하지 않는가
- [ ] 영향을 받는 도메인과 담당자를 Issue의 `관련 작업`에 적었는가
- [ ] 선행·후속 Issue 또는 병합 순서를 확인했는가
- [ ] API 요청·응답·오류 계약이 바뀌는가
- [ ] Entity·DB 컬럼·제약조건·인덱스가 바뀌는가
- [ ] 예약·참여자·결제 상태 전이가 바뀌는가
- [ ] 트랜잭션 실패 후 남는 데이터가 바뀌는가
- [ ] 동시 요청 결과와 중복 방지 방식이 바뀌는가
- [ ] 권한과 소유권 검증이 바뀌는가
- [ ] 환불·지급 예정금 계산이 바뀌는가
- [ ] 채팅 접근 권한이 바뀌는가
- [ ] 전체 플로우차트·시퀀스·API·ERD 중 실제 영향 문서만 수정했는가
- [ ] 완료 조건과 테스트가 변경된 계약을 증명하는가
- [ ] 미확정 정책을 구현 범위에서 제외했는가

기존 플로우차트와 상태도를 이 문서에 복사하지 않는다. 변경된 정책의 영향 위치만 연결한다.

## 6. 현재 결정 필요 항목

다음 항목은 확정 정책으로 사용하지 않는다. 관련 기능 Issue를 `READY`로 전환하기 전에 팀 결정이 필요하다.

| 결정 필요 항목 | 결정 시점 | 영향 도메인 |
|---|---|---|
| 식사 시작 2시간 이후 최초 예약자 취소 | 취소 기능 구현 전 | 예약, 환불, 좌석, 회차, 채팅 |
| 결제 성공 후 예약·참여자 저장 실패 보상 | 예약·결제 통합 전 | 결제, 예약, 좌석 |
| 마지막 자리 동시 결제 결과 | 동시성 구현 전 | 결제, 예약, 좌석 |
| 동일 사용자의 동일 시간대 중복 참여 | 예약 API 확정 전 | 예약, 검색, 인증 |
| 방문·노쇼 처리 시작·마감 시간 | v2 노쇼 구현 전 | 사장님, 참여자, 예약 |
| 잘못된 방문·노쇼 상태 정정 | v2 노쇼 구현 전 | 상태 이력, 지급 예정금, 노쇼율 |
| 미처리 참여자와 `CLOSED` 전환 방식 | v2 노쇼 구현 전 | 예약, 참여자, 지급 예정금 |
| 채팅방 생성 시점과 `RECRUITING` 접근 | v2 채팅 구현 전 | 예약, 채팅 |
| 취소·종료 후 채팅 읽기·쓰기 권한 | v2 채팅 구현 전 | 취소, 예약, 채팅 |
| 채팅 메시지 저장·보관 기간 | v2 채팅 구현 전 | 채팅, 개인정보 |
| 실제 환불 처리 방식 | v2 환불 구현 전 | 결제, 환불 |

노쇼 이의 제기, 실제 계좌 송금, 재정산 시스템은 현재 범위에 추가하지 않는다.
